# 은퇴월급 — 설계 스펙 (2026-07-09 확정)

> 브레인스토밍 확정본. 대장님 승인: 정체성·하이브리드 아키텍처·MVP 3화면·이름·무료+광고 모델.

## 1. 정체성

**"다음 달 내 은퇴 월급은 얼마?"** — 보유 종목 배당 + 연금 인출 계획을 합산해 월별 현금흐름을
달력으로 보여주고, 세금 임계치 3종에 도달하기 전 게이지로 경고하는 앱.

- 타깃: 한국 은퇴자·예비은퇴자·배당 투자자 (연금나침반 유저의 다음 질문)
- 스토어명: **`은퇴월급: 배당·연금 현금흐름 달력 & 세금 경보`** (26자, 30자 제한 내)
- 패키지(예정): `com.quantlog.retirepaycheck` / repo: `~/Workspace/retire-paycheck`

## 2. 시장 근거 (2026-07-09 Play·App Store 조사)

- **세금 임계치 게이지 = 양 스토어 순수 공백** (국내외 전무) — 유일 기능
- 배당+연금 합산 달력: 근접 경쟁 **이자(eZar)** 유일 (계좌연동 기반, 월 9,900원 구독, 평점 4.7~4.8)
- "은퇴 월급" 키워드: 콘텐츠 수요 多·앱 공급 0 — ASO 선점
- **포지셔닝 (이자의 정반대)**: "계좌 연동 없이 3분 입력 · 내 자산정보는 폰 밖으로 안 나감 · 무료"
- 전략: 이자가 임계치 기능을 얹기 전 선점하는 **속도전** — MVP 최소 컷 유지

## 4. MVP 3화면

### 화면 1 — 자산 입력
- 보유 종목: 종목명 검색(로컬 종목 사전 or /dividends의 corp 목록 활용) + 보유 수량
- 연금 계좌: 연금저축 / IRP / ISA 잔액 3필드 (연금나침반 간단판) + 현재 나이
- 월 인출 계획 **2필드 분리**: ①연금저축·IRP에서 월 ○○만원 (전액 과세대상 **보수 가정** — 1,500만
  게이지 산입) ②ISA·기타에서 월 ○○만원 (비과세 취급). 유저가 아는 "어느 계좌에서 빼는지" 기준으로
  받고, 공제분/비공제분 정밀 구분은 연금나침반 CTA로 유도 (시리즈 시너지)
- (선택) 연간 이자소득 수동 입력 — 금융소득종합과세 게이지 정확도용
- 저장은 전부 온디바이스

### 화면 2 — 은퇴 월급 달력 (메인)
- 월별 뷰: 배당 입금 예정(확정 = 진하게 / 예측 = 흐리게+"예상" 배지) + 연금 인출 계획액 합산
- 상단 카드 **세후로 통일**: 메인 = "이번 달 실수령 ○○○만원" (배당 15.4% 원천징수 기준 +
  연금 나이별 연금소득세 반영 — 나침반 엔진 재사용), 서브로 "세전 ○○○만원" 병기.
  ⚠️ 세전·세후 혼합 지표 금지 (대표 숫자 신뢰가 앱의 생명)
- 배당액 = per_share × 보유수량. **예상 지급월 휴리스틱 명문화**: 결산배당(12월 말 기준일) →
  익년 4월 / 분기배당(3·6·9월 말 기준일) → 기준일 익월 말. 라벨은 항상 "예상 지급월" +
  "회사별로 지급 시기가 다를 수 있음" 고지 (단정 표기 금지)

### 화면 3 — 세금 임계치 게이지 3종
| 게이지 | 기준 | 산식(연간 누적) |
|---|---|---|
| 금융소득종합과세 | 2,000만원 | (연간 배당 예상 합계 + 입력한 이자소득) ÷ 2,000만 |
| 건강보험 피부양자 | 연 소득 2,000만원 (상수) | (금융소득+연금소득 등 합산) ÷ 2,000만 — **"참고용" 배지 필수** + "공적연금·기타 소득 합산에 따라 달라질 수 있음" 디스클레이머. 산입 규칙(금융소득 연 1,000만 초과 시 전액 산입 등) 구현 전 법령 확인 |
| 사적연금 저율한도 | 1,500만원 | 연금저축·IRP 과세대상 연 인출 계획 ÷ 1,500만 (연금나침반 엔진 로직) |
- 80% 이상 노랑, 100% 초과 빨강 + 초과 시 한 줄 설명("초과 시 전액 16.5% 전환" 등)
- 푸시 없음 — 앱 내 게이지만 (P2)
