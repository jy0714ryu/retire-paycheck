# 은퇴월급 — 설계 스펙 (2026-07-09 확정)

> 브레인스토밍 확정본. 대장님 승인: 정체성·하이브리드 아키텍처·MVP 3화면·이름·무료+광고 모델.

## 1. 정체성

**"다음 달 내 은퇴 월급은 얼마?"** — 보유 종목 배당 + 연금 인출 계획을 합산해 월별 현금흐름을
달력으로 보여주고, 세금 임계치 3종에 도달하기 전 게이지로 경고하는 앱.

- 타깃: 한국 은퇴자·예비은퇴자·배당 투자자 (연금나침반 유저의 다음 질문)
- 스토어명: **`은퇴월급: 배당·연금 현금흐름 달력 & 세금 경보`** (26자, 30자 제한 내)
- 패키지(예정): `com.quantlog.retirepaycheck` / repo: `~/Workspace/retire-paycheck`

## 2. 시장 근거 (2026-07-09 Play·App Store 조사)

- **세금 임계치 게이지 = 양 스토어 순수 공백** (국내외 전무) — 유일 기능
- 배당+연금 합산 달력: 근접 경쟁 **이자(eZar)** 유일 (계좌연동 기반, 월 9,900원 구독, 평점 4.7~4.8)
- "은퇴 월급" 키워드: 콘텐츠 수요 多·앱 공급 0 — ASO 선점
- **포지셔닝 (이자의 정반대)**: "계좌 연동 없이 3분 입력 · 내 자산정보는 폰 밖으로 안 나감 · 무료"
- 전략: 이자가 임계치 기능을 얹기 전 선점하는 **속도전** — MVP 최소 컷 유지

## 3. 아키텍처 — 하이브리드

```
[온디바이스 (SharedPreferences)]          [맥미니 공개 API — 읽기전용]
보유종목·수량 / 연금 계좌 잔액             https://api.quant-view.co.kr
월 인출 계획 / (선택) 연 이자소득           └ GET /dividends  ← 실측 확인(2026-07-09 200 OK)
연금나침반 세금 엔진(복사) 계산               스키마: corp_name, corp_code, ex_date,
임계치 게이지 계산                                  record_date, per_share, is_confirmed,
                                                    source, disclosed_at
전송되는 것 = 없음 (배당 조회도 전체 리스트 수신 후 로컬 필터)
오프라인 = 마지막 응답 캐시로 동작
```

- 개인 자산정보 서버 전송 0 — 프라이버시가 기술 선택이자 **마케팅 무기**
- Flutter, `pension-compass/app` 구조 복제 (Riverpod·fl_chart·SharedPreferences·AdMob 패턴)
- 재사용 자산: ① 연금나침반 `pension_simulator.dart`/`tax_calculator.dart` (파일 복사+출처 주석)
  ② 공시한줄 배당 데이터 (확정+예측, `is_confirmed` 구분) ③ AdMob·서명·게시 파이프라인 노하우

## 4. MVP 3화면

### 화면 1 — 자산 입력
- 보유 종목: 종목명 검색(로컬 종목 사전 or /dividends의 corp 목록 활용) + 보유 수량
- 연금 계좌: 연금저축 / IRP / ISA 잔액 3필드 (연금나침반 간단판) + 월 인출 계획액 + 현재 나이
- (선택) 연간 이자소득 수동 입력 — 금융소득종합과세 게이지 정확도용
- 저장은 전부 온디바이스

### 화면 2 — 은퇴 월급 달력 (메인)
- 월별 뷰: 배당 입금 예정(확정 = 진하게 / 예측 = 흐리게+"예상" 배지) + 연금 인출 계획액 합산
- 상단 카드: "이번 달 은퇴 월급 ○○○만원" (배당 세후 15.4% 반영, 연금은 인출 계획 그대로)
- 배당액 = per_share × 보유수량, 지급월 = record_date 기준 통상 지급 시차 반영(단순 규칙, 주석 명시)

### 화면 3 — 세금 임계치 게이지 3종
| 게이지 | 기준 | 산식(연간 누적) |
|---|---|---|
| 금융소득종합과세 | 2,000만원 | (연간 배당 예상 합계 + 입력한 이자소득) ÷ 2,000만 |
| 건강보험 피부양자 | 소득 기준 | (배당+이자+연금소득 등 합산) ÷ 기준값 — **정확 기준값·산입 범위 구현 전 법령 확인 필수** |
| 사적연금 저율한도 | 1,500만원 | 연금저축·IRP 과세대상 연 인출 계획 ÷ 1,500만 (연금나침반 엔진 로직) |
- 80% 이상 노랑, 100% 초과 빨강 + 초과 시 한 줄 설명("초과 시 전액 16.5% 전환" 등)
- 푸시 없음 — 앱 내 게이지만 (P2)

## 5. 수익 모델
- **MVP 유료화 없음** — 무료 + AdMob (배너 + 전면 2회당 1회, 연금나침반 코드 복제, 광고단위 신규 발급)
- 후속 수익화 후보 (지표 확인 후, 인앱결제 추가는 언제든 가능): 광고 제거 / 시나리오 복수 저장 / 연간 세금 리포트

## 6. 제외 범위 (YAGNI — MVP에 넣지 않음)
푸시 알림(FCM·서버) / 로그인·계정 / 증권사·마이데이터 연동 / 해외 배당 / iOS(Android 반응 후) / 위젯

## 7. 구현 전 확인 사항 (플랜 단계 반영)
1. `/dividends` 응답 전체 건수·용량 — 크면 gongsi 백엔드에 `?corp_codes=` 필터 파라미터 1개 추가
2. 건보 피부양자 소득 기준 2026년 정확값·산입 항목 (법령 확인 → 상수 확정, UI에 기준일 명시)
3. 종목명 검색 사전 소스 (corp_code↔종목명 — gongsi corpcode 재사용 or /dividends 누적으로 충분한지)
4. 배당 지급일 규칙 (record_date → 실제 입금월 시차) 단순화 수준 결정

## 8. 시리즈 연계
- 상호 CTA: 연금나침반(인출 순서 최적화) ↔ 은퇴월급(월별 현금흐름) ↔ 공시한줄(공시·배당 알림)
- 블로그 앵커 글 패턴 재사용: "은퇴 월급" 키워드 콘텐츠 공백 선점

## 9. 리스크
- 이자(eZar)의 기능 확장 — 속도로 대응
- 배당 예측(is_confirmed=false)의 정확도 — "예상" 배지로 기대 관리, 단정 표기 금지
- 건보 기준 등 제도 변경 — 상수 중앙화 + 기준일 표기
