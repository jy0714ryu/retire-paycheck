## Important

### I1. [수정완료] 저장 UX — 자동저장 동작하나 피드백 전무
- **판정:** `HoldingsNotifier`·`RetirementInputNotifier` 는 값 변경 즉시 `_persist()`로
  SharedPreferences 에 저장(fire-and-forget), 생성자에서 `_load()`로 복원. **자동저장 맞음.**
- **실기기 검증:** 대장님 직전 세션 입력(삼성전자 12주·연금 1,000만·인출값)이 `am force-stop`
  후 재기동에도 그대로 유지됨 → 자동저장·복원 정상. (면책 동의 플래그도 영속 확인.)
- **문제:** 별도 저장 버튼이 없고 저장 피드백이 없어 "저장됐나?" 불안(대장님 지적).
- **조치:** 입력 화면 상단에 상시 안내 배너 추가 —
  "입력한 내용은 자동으로 저장됩니다. 앱을 닫아도 값이 유지돼요."(초록 cloud-done 아이콘).
  키 입력마다 뜨는 SnackBar 대신 상시 고지 채택(가볍고 확실, 잦은 토스트 소음 회피). 회귀 테스트 추가.

### I2. [문서만] 사적연금 저율한도 초과 시 net 계산이 16.5%를 반영하지 않음
- `CashflowEngine.buildMonths` 의 연금 net = 항상 `pensionTaxRate(age)`(70세 미만 5.5%) 적용.
  그러나 사적연금 연 인출이 1,500만원 초과하면 **전액 16.5% 종합/분리과세로 전환**(게이지가 경고).
- **불일치:** 인출을 월 125만 초과로 올리면 게이지는 빨강(>100%)이 되지만, 달력 헤드라인
  "이번 달 실수령"은 여전히 5.5% 기준이라 **실수령을 과대 표시**한다.
- **제안(저비용 아님 — 세무 모델 결정 필요):** buildMonths 에서 `monthlyPensionWithdrawal*12 >
  kPensionLowRateLimit` 이면 세율을 0.165로 스위치. 단, 월 뷰가 연 누계를 인지하도록 배선 필요 +
  회귀 테스트. 저율/고율 경계는 대장님이 게이지로 의도적으로 분리해둔 영역이라 **직접 수정 보류**,
  판단 요청.

### I3. [문서만] 배당 데이터가 전량 예측 + 대부분 12월 기준일 → 현재 달력엔 배당이 거의 0
- API 실측(1,000건): `is_confirmed` 전부 false(예측), 992/1,000이 record_date 2026-12 →
  휴리스틱상 **2027-04 지급**. 삼성전자도 per_share 1,668·12월 기준.
- **영향:** 2026년 달력 대부분의 달은 배당 0으로 표시(가치가 4월 한 달에 몰림), 그리고 모든 배당
  라인이 상시 "예상"(흐린 회색) 배지 → 배지 변별력 저하·"앱이 비어 보임" 인상.
- 코드 결함 아님(데이터 특성). 완화책은 D1(연간 예상 배당 요약) 참조.
