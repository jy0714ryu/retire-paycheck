# 은퇴월급 v2 설계 — 계좌 유형별 세금 정확도

- 작성: 2026-07-11 (브레인스토밍 확정, Gemini 교차 리뷰 반영)
- 상태: 스펙 확정 대기 (구현 착수는 Play 심사 통과·v1.1.0+2 제출 이후)
- 근거: 대장님 제안 6건 + 경쟁앱 10종 리뷰 분석 (요구 4위 "ISA·연금저축 절세효과 수치화"가 유일한 기능 공백)

## 0. 테마와 원칙

**테마: "계좌 유형이 곧 세금이다."** 같은 배당이라도 일반계좌(15.4%+종합과세 산입) /
ISA(과세이연·미산입) / 연금계좌(과세이연·계좌 내 재투자)로 처리가 갈린다.
v1은 전부 일반계좌 취급 → ISA·연금계좌 보유자에게 금융소득 게이지가 **과대 경고**되는
정확도 결함이 있다. v2는 이를 수정하는 릴리스다.

**유지 원칙:** 3분 입력 · 온디바이스(자산 정보 폰 밖 전송 금지) · 계좌 연동 없음 · 무료.

**제외 (YAGNI):** 매입가·수익률 트래킹(더리치 본진, 정체성 이탈) / 과거 수령 이력(v3) /
목표 생활비(미선택) / 평생권 IAP(v1.2 별도 릴리스) / 배당락 알림.

## 1. 데이터 모델

### 1.1 신규 `Account`

```dart
enum AccountType { general, isa, pension }

class Account {
  final String id;       // uuid 또는 'default_<type>'
  final String name;     // 예: "미래에셋 ISA"
  final AccountType type;
}
```

- SharedPreferences `accounts` 키에 jsonEncode 리스트로 영속.
- **기본 계좌 3개는 암묵 존재** — id `default_general` / `default_isa` / `default_pension`,
  이름 "일반계좌"/"ISA"/"연금계좌". 저장소에 기록하지 않고 코드 상수로 항상 제공
  (저장소 로드 순서 의존성 제거). 삭제·이름변경 불가.
- 유저 생성 계좌는 이름+유형 필수. 유형 변경 불가(세금 이력 꼬임 방지) — 잘못 만들면
  삭제 후 재생성. 계좌 삭제 시 소속 종목은 해당 유형의 기본 계좌로 이동(종목 삭제 아님).

### 1.2 `Holding` 확장

- `String accountId` 필드 추가 (기본값 `default_general`).
- toJson: `account_id` 키 기록. fromJson: 키 없으면 `default_general`
  (v1 저장분 자동 마이그레이션 — 지금까지 전부 일반계좌 취급이었으므로 동작 불변).
- 수동 입력 종목(manual_*)도 동일하게 accountId 부여.

### 1.3 `RetirementInput` 확장

- `bool isWithdrawing` 추가 — 연금 인출 모드 토글.
- 마이그레이션: 필드 없으면 `monthlyPensionWithdrawal > 0 || monthlyOtherWithdrawal > 0`
  → true, 아니면 false. (오분류 가능성은 토글로 즉시 수정 가능 — 입력 화면 캡션 안내)

### 1.4 신규 `InterestItem`

```dart
class InterestItem {
  final String id;
  final String name;          // 예: "새마을금고 정기예금"
  final int annualAmount;     // 연 이자 금액(원, 세전)
  final List<int> months;     // 지급월. 빈 리스트 = 월 균등(12분할)
}
```

- SharedPreferences `interest_items` 키. 이자율×원금 계산은 지원하지 않는다
  (입력 2배 — 유저는 만기 안내문으로 연 이자 금액을 이미 안다).

### 1.5 마이그레이션 멱등성 (★ Gemini 질문에서 도출)

- SharedPreferences `schema_version` 키(int, v1 저장분 = 키 없음 = 1).
- 앱 기동 시 `schema_version < 2` 이면 1회 실행:
  1. `annualInterestIncome > 0` → InterestItem("이자소득", 금액, 월 균등) 1개 생성.
  2. `isWithdrawing` 판정(1.3 규칙) 후 저장.
  3. `schema_version = 2` 기록.
- **구 필드(`annual_interest_income`)는 지우지 않는다** — 구버전 롤백 시 이자 유실 방지.
  구버전 fromJson 은 모르는 키를 무시하므로 신→구→신 왕복 안전. 재마이그레이션은
  schema_version 가드로 차단(이자 항목 중복 생성 금지).
- 알려진 한계(문서화만): 구버전 롤백 상태에서 저장하면 `account_id` 가 유실되어
  재업데이트 시 종목이 일반계좌로 복귀한다. 발생 확률 낮음 — 게이지 과대 경고
  방향(안전측)이므로 수용.

## 2. 세금 엔진 분기 (cashflow_engine)

| 계좌 유형 | 배당 달력 처리 | 금융소득 게이지 | 건보 게이지 |
|---|---|---|---|
| 일반 | 세후 15.4% → "실수령" 합산 | **산입** (gross) | **산입** (gross) |
| ISA | 세금 0(과세이연·손익통산) → 실수령 합산, "ISA" 라벨 | **미산입** | **미산입** |
| 연금계좌 | 통장 미입금 → 실수령 제외, 하단 "계좌 내 재투자" 섹션 | 미산입 | 미산입 |

- **ISA·연금계좌 배당은 건강보험 피부양자 게이지에서도 미산입** — 비과세·과세이연
  소득은 건보 소득 산정에서 제외된다(현행 기준). 금융소득 게이지와 동일 분기 지점을
  사용해 로직 이원화 금지. (참고: 사적연금 인출도 기존 결정대로 건보 미산입 유지)
- **ISA 절세효과** = 연간 ISA 배당 gross × 15.4%. "일반계좌 대비 연 XX만원 절세 중"
  카드로 표시, 캡션 "(비과세 한도 내 기준)" 고정 — 한도 초과 9.9% 분리과세는 계산하지
  않는다(만기·서민형 여부를 앱이 모름).
- **이자소득**: 항목별 세후 15.4%를 지급월(또는 12분할)에 달력 반영. 게이지에는
  gross 산입(금융소득·건보 모두).
- **연금 모드 OFF**: 인출 0 취급 — 달력에 연금 타일 없음, 연금소득세·1,500만 절벽
  게이지 비활성(참고 표시), 연금계좌 종목 배당은 재투자 섹션만.
- 재투자 섹션 금액은 세금 0 (과세이연 — 인출 시점 과세는 인출 모드의 연금소득세가 담당).

## 3. 화면

### 입력 탭
- 종목 추가 바텀시트: 계좌 선택 = 유형 3버튼(기본, 1탭). 유저 계좌가 2개 이상인
  유형은 버튼 탭 시 계좌 목록 노출.
- "계좌 관리" 접힌 섹션: 계좌 추가(이름+유형)·삭제. 기본 계좌는 목록에 항상 표시.
- 연금 섹션: "연금 인출 중" 토글. OFF 시 인출 입력란 숨김.
  토글 옆 작은 텍스트 CTA: **"어떤 순서로 빼야 세금을 아낄까? → 연금나침반"**
  (Play 스토어 링크. 방해성 금지 — 다이얼로그·팝업 아님, 한 줄 링크만)
- 이자소득: InterestItem 리스트 편집 (추가/삭제, 이름·금액·지급월).

### 달력 탭
- **상단 요약 카드 = 항상 최종 합산 고정** (배당+연금+이자 실수령. 필터와 무관 — 대장님 지시).
- 그 아래 필터 칩: `[전체 | 일반 | ISA | 연금]` + 유저 계좌 2개 이상 시 계좌명 칩 추가
  (모아보기/계좌별 보기 전환). 필터는 월별 상세·연간 차트에만 적용.
- **"계좌 내 재투자" 섹션: 기본 접힘(Accordion)·보조 텍스트 톤** — 메인 "실수령"
  직관성을 해치지 않는다. 펼치면 연금계좌 배당 내역 + "과세이연·인출 전까지 계좌 내
  재투자" 캡션.

### 게이지 탭
- 금융소득·건보 게이지: 분기 산식 반영 (일반계좌 배당+이자만 산입).
- ISA 절세효과 카드 추가.

## 4. 마케팅 (코드 외 산출물)

- 스토어 설명·릴리스 노트·앵커글 반영 카피:
  - "로그인도 계좌 연동도 없습니다 — 자산 정보는 폰 밖으로 나가지 않습니다"
    (경쟁앱 실불만: 마이데이터 중복동의·연동 끊김·개인정보 부담)
  - "틀린 수량? 그 자리에서 바로 고치세요" (연동 앱 반영 지연 2~3일 대비)
  - "ISA·연금계좌까지 구분하는 세금 계산 — 절세액이 눈에 보입니다"
- 연금나침반 ↔ 은퇴월급 상호 CTA (3절 입력 탭 + 연금나침반 측은 별도 작업).

## 5. 검증

- 단위테스트: 엔진 유형별 분기(3유형×달력/게이지 2종), ISA 절세효과 계산,
  이자 지급월/12분할, 연금 모드 ON/OFF, 마이그레이션 3종(accountId 기본값·
  이자 변환·isWithdrawing 판정) + 멱등성(schema_version 2회 기동 중복 생성 없음).
- 위젯테스트: 필터 칩 전환, 재투자 아코디언, 계좌 선택 바텀시트.
- 릴리스 빌드 실기기 E2E 필수 (v1 교훈: INTERNET 권한·lazy provider 는 debug 로 못 잡는다).
